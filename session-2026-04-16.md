### Changes made this session

#### 1. Renamed `printXxxWithOutput` → `printXxx`

The 5 new print functions had been given a `WithOutput` suffix that is inconsistent with `deployment-package.go` which just uses `printDeploymentPackages`.  Renamed all 5:

| Old name | New name |
|---|---|
| `printApplicationsWithOutput` | `printApplications` |
| `printArtifactsWithOutput` | `printArtifacts` |
| `printDeploymentsWithOutput` | `printDeployments` |
| `printDeploymentProfilesWithOutput` | `printDeploymentProfiles` |
| `printProfilesWithOutput` | `printProfiles` |

All call sites updated accordingly.

#### 2. Deleted dead old `printXxxSimple` / `printXxx` functions and header vars

The old pre-pipeline print functions and their `var xxxHeader` vars were left as dead code alongside the new ones.  Removed:

- `application.go`: `var applicationHeader`, `func printApplicationsSimple`
- `artifact.go`: `var artifactHeader`, `func printArtifactsSimple` (was still called from `printArtifactEvent` — inlined its logic directly there)
- `deployment.go`: `var deploymentHeader`, `func printDeployments` (old simple one)
- `deployment-profile.go`: `var deploymentProfileHeader`, `func printDeploymentProfiles` (old simple one)
- `profile.go`: `var profileHeader`, `func printProfiles` (old simple one)

#### 3. Removed unnecessary intermediate output row structs

Each of the 5 files had been given an intermediate `type xxxOutputRow struct` + `func toXxxOutputRows()` that pre-converted API fields before templating.  `deployment-package.go` has none of these — it passes the raw API slice straight as `Data` and lets the template handle nil pointer fields with the `str`/`deref` template funcs.

Removed:
- `type applicationOutputRow struct` + `func toApplicationOutputRows()`
- `type artifactOutputRow struct` + `func toArtifactOutputRows()`
- `type deploymentOutputRow struct` + `func toDeploymentOutputRows()`
- `type deploymentProfileOutputRow struct` + `func toDeploymentProfileOutputRows()`
- `type profileOutputRow struct` + `func toProfileOutputRows()`

Also removed the `if outputType == "table" { data = toXxxOutputRows(...) }` branching from each `printXxx` function, and the `b64 "encoding/base64"` import from `profile.go` (was only used in `toProfileOutputRows`).

#### 4. Updated templates to use raw API fields

With the structs removed, templates were updated to reference the raw API struct fields using the `str`/`deref` template funcs:

**application.go**
- Inspect format: `{{str .DisplayName}}`, `{{str .Description}}`, `{{str .ImageRegistryName}}`, `{{str .DefaultProfileName}}`
- Profiles list: `{{- range deref .Profiles}}\n  {{.Name}}{{- end}}`

**artifact.go**
- Inspect format: `{{str .DisplayName}}`, `{{str .Description}}`

**deployment.go**
- List format changed from `{{.DeploymentID}}\t{{.Name}}\t{{.DisplayName}}\t{{.Profile}}\t{{.State}}` (struct fields) to `{{str .DeployId}}\t{{str .Name}}\t{{str .DisplayName}}\t{{str .ProfileName}}\t{{.Status.State}}` (raw API fields)
- Inspect format same field renames

**deployment-profile.go**
- List format: `{{len .ApplicationProfiles}}` instead of `{{.ProfileCount}}`
- Inspect format: `{{str .DisplayName}}`, `{{str .Description}}`, replaced `{{.Profiles}}` string with proper range: `{{range $app, $profile := .ApplicationProfiles}}\n  {{$app}}:{{$profile}}{{- end}}`

**profile.go**
- Inspect format: `{{str .DisplayName}}`, `{{str .Description}}`, `{{str .ChartValues}}`
- Deployment requirements: `{{- range deref .DeploymentRequirement}}\n  {{.Name}}:{{.Version}}{{if .DeploymentProfileName}}:{{str .DeploymentProfileName}}{{end}}{{- end}}`
- Parameter templates: `{{- range deref .ParameterTemplates}}\n  {{.Name}}={{.Type}}:{{str .DisplayName}}:{{str .Default}}{{- end}}`

## Current state of the print functions (all 5 match deployment-package.go pattern)

Each `printXxx` function now:
1. Gets `output-type` flag
2. Calls `getXxxOutputFormat(cmd, verbose)` which either returns inspect format (verbose) or resolves table template from flag/envvar
3. Builds `CommandResult{Format, Filter, OrderBy, OutputAs, NameLimit, Data: *rawAPISlice}`
4. Calls `GenerateOutput(writer, &result)`

No intermediate structs, no data switching.

## Template functions available
- `str .Field` — nil-safe `*string` deref (returns `""` for nil)
- `deref .Field` — nil-safe pointer deref for any type (returns zero value for nil)
- `timestamp .Field` — formats a `*timestamppb.Timestamp` (not applicable here; fields are `*time.Time`)

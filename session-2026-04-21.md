### 9. Added Template Helper Functions (`pkg/format/formatter_funcs.go`)

**New Functions:**

#### `formatStatusIndicator(status interface{}) string`
- Extracts indicator from GenericStatus objects
- Returns symbols: ✓ ⚠ ⨯ ⏳ ?
- Uses reflection for flexibility
- Template name: `statusIndicator`

**Indicator Mapping:**
- `STATUS_INDICATOR_IDLE` → ✓ (ready)
- `STATUS_INDICATOR_WARN` → ⚠ (warning)
- `STATUS_INDICATOR_ERROR` → ⨯ (error)
- `STATUS_INDICATOR_INFO` → ⏳ (in progress)
- Unknown/nil → ? (unknown)

#### `formatStatusMessage(status interface{}) string`
- Extracts message from GenericStatus objects
- Returns message string or "<unknown>"
- Uses reflection for flexibility
- Template name: `statusMessage`

#### `formatNodeCount(count *int) string`
- Formats node count or returns "-" if nil
- Template name: `nodeCount`

**Benefits:**
- Clean, readable cluster status display
- Reusable for other resources with GenericStatus
- Handles nil values gracefully

## Example Usage

### List Clusters with Different Output Types

```bash
# Table output (default)
./build/_output/orch-cli list cluster --project itep

# JSON output
./build/_output/orch-cli list cluster --project itep --output-type json

# YAML output
./build/_output/orch-cli list cluster --project itep --output-type yaml
```

### Ordering Clusters

```bash
# Order by name (ascending)
./build/_output/orch-cli list cluster --project itep --order-by name

# Order by name (descending)
./build/_output/orch-cli list cluster --project itep --order-by -name

# Case-insensitive field names
./build/_output/orch-cli list cluster --project itep --order-by kubernetes_version
./build/_output/orch-cli list cluster --project itep --order-by kubernetesVersion
```

### Filtering Clusters

```bash
# API-level filter
./build/_output/orch-cli list cluster --project itep --filter 'name~test'

# Client-side filter
./build/_output/orch-cli list cluster --project itep --output-filter 'KubernetesVersion=1.28.0'

# Show only not-ready clusters
./build/_output/orch-cli list cluster --project itep --not-ready

# Combine filters
./build/_output/orch-cli list cluster --project itep --not-ready --output-filter 'Name~prod'
```

### Custom Templates

```bash
# Custom inline template
./build/_output/orch-cli list cluster --project itep \
  --output-template '{{.Name}} ({{.KubernetesVersion}})'

# Template from file
./build/_output/orch-cli list cluster --project itep \
  --output-template-file my-cluster-template.txt

# Environment variable
export ORCH_CLI_CLUSTER_OUTPUT_TEMPLATE='{{.Name}}\t{{.KubernetesVersion}}\t{{statusMessage .LifecyclePhase}}'
./build/_output/orch-cli list cluster --project itep
```

### Verbose Output

```bash
# Get detailed cluster information
./build/_output/orch-cli get cluster my-cluster --project itep --verbose
```

**Output:**
```

Name: my-cluster

  - ID: host-abc123, Role: control-plane

  Lifecycle Phase: Cluster is provisioned
  Provider Status: Provider is ready
  Control Plane Ready: Control plane is ready
  Infrastructure Ready: Infrastructure is ready
  Node Health: All nodes are healthy

  environment: production
```

FIXES APPLIED
-------------

Fix 1 — KVM MPS relay TLS minimum version (SDL430-12)
  Before: tls.Config{MinVersion: tls.VersionTLS12, ...}
  After:  tls.Config{MinVersion: tls.VersionTLS13, ...}
  Reason: TLS 1.2 permits cipher suites with known weaknesses (e.g. CBC,
          RC4 fallback). TLS 1.3 eliminates these and aligns KVM with the
          SoL implementation which already used TLS 1.3 on the same MPS.
  File:   internal/cli/kvm_viewer.go, mpsRelayTLSConfig()

Fix 2 — KVM WebSocket origin validation (SDL523-13)
  Before: CheckOrigin: func(_ *http.Request) bool { return true }
  After:  CheckOrigin validates that Origin header (when present) has
          hostname == "localhost" or "127.0.0.1"
  Reason: Defense-in-depth. Although the server only listens on 127.0.0.1
          and all routes require the session token, accepting any Origin
          violates the principle of minimal permissions. A browser script
          from a different origin cannot obtain the session token (it is
          injected into the orch-cli-launched tab only), but explicit
          rejection at the WebSocket upgrade layer removes the surface.
  File:   internal/cli/kvm_viewer.go, startKVMViewer()


VERIFICATION COMMANDS
---------------------
The following commands can be run after starting a KVM session to confirm
minimal permissions. Replace <PORT> with the port printed at session start.

1. Confirm server is bound to loopback only (not 0.0.0.0):
   ss -tlnp | grep <PORT>
   Expected: Local Address = 127.0.0.1:<PORT>

2. Confirm network interface is NOT listening externally:
   ss -tlnp | grep <PORT> | grep -v 127.0.0.1
   Expected: no output

3. Confirm TLS 1.3 on outbound MPS connection:
   (logged at session start)
   [KVM] TLS: using CA certificate from "orch-ca.crt"

4. Confirm token rejection (403) without valid token:
   curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:<PORT>/api/status
   Expected: 403

5. Confirm WebSocket upgrade rejected from non-localhost origin:
   curl -s -o /dev/null -w "%{http_code}" \
     -H "Origin: http://evil.example.com" \
     -H "Upgrade: websocket" \
     -H "Connection: Upgrade" \
     -H "Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==" \
     -H "Sec-WebSocket-Version: 13" \
     "http://127.0.0.1:<PORT>/ws/kvm?token=<TOKEN>"
   Expected: 403

6. Confirm no named pipes or Unix sockets:
   grep -rn "ListenUnix\|mkfifo\|os\.Pipe" \
     internal/cli/kvm_viewer.go internal/cli/sol_session.go
   Expected: no output


CONCLUSION
----------
SDL430-12: CLOSED — All ports use minimal binding (loopback+ephemeral),
           TLS 1.3 minimum enforced, token authentication on all routes.
SDL523-13: CLOSED — No excessive permissions. WebSocket origin validated.
           File writes use mode 0600. No privileged ports or pipes.

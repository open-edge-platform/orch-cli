#### 8. Implemented case-insensitive field name support for `--order-by`

**Issue:** User discovered deployment-packages accept `--order-by name` (lowercase) while other resources require `--order-by Name` (PascalCase).

**Root cause:** Deployment-packages use `normalizeOrderByForClientSorting()` which builds field name aliases (camelCase, snake_case, lowercase all work). Other resources passed raw flag value without normalization.

**Fix:** Added `normalizeOrderByForClientSorting()` validation to all resources:

**Files updated:**
- `internal/cli/application.go` (lines 258-272, 289-291, 350-352)
- `internal/cli/artifact.go` (lines 173-187, 203-205, 263-265)
- `internal/cli/deployment.go` (lines 431-445, 461-463, 520-522)
- `internal/cli/profile.go` (lines 325-337)

**Pattern implemented:**
```go
// Validate and normalize order-by for client-side sorting (table output)
var clientOrderBy *string
if outputType == "table" {
    if apiOrderBy != nil {
        var sampleStruct SomeAPIType
        clientOrderBy, err = normalizeOrderByForClientSorting(*apiOrderBy, sampleStruct)
        if err != nil {
            return err
        }
    }
    apiOrderBy = nil  // Client-side sorting for table
}
```

**Benefits:**
- All field name formats now work: `Name`, `name`, `displayName`, `display_name`, `DISPLAY_NAME`
- Helpful error messages show available fields: `invalid --order-by field "badField"; available fields: chartName, chartVersion, createTime, ...`
- Consistent behavior across all resources (applications, artifacts, deployments, profiles, deployment-packages)

**Verified working:**

## Discovered but not fixed

### Issue: Artifacts `--filter` flag causes API error
**Symptom:** Using `--filter` on artifacts results in "error listing artifacts:[400 Bad Request]"

**Root cause:** The artifacts API endpoint does not support server-side filtering, but the CLI passes the filter parameter anyway.

**Impact:** Users attempting to use `--filter` with artifacts get an unhelpful error. They should use `--output-filter` instead.

**Potential fixes:**
1. Remove `--filter` flag from artifacts entirely (only keep `--output-filter`)
2. Keep `--filter` flag but don't pass it to API (treat as alias for `--output-filter`)
3. Investigate if API could/should support filtering

**Status:** Documented in todoDocs.md, not yet implemented
